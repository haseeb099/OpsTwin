# Contributing to OpsTwin

Thank you for your interest in contributing. Contributions of all kinds are welcome — bug fixes, new features, documentation improvements, and more.

---

## Getting Started

1. **Fork** the repository and create a feature branch from `main`:
   ```bash
   git checkout -b feat/your-feature-name
   ```

2. Make your changes. Follow the existing code style and keep commits focused.

3. Open a Pull Request against `main` with a clear description of what the PR does and why.

---

## PR Requirements

Every pull request must include the following:

1. **Cursor task template** — paste the `.cursor/task-template.md` you used to generate or guide the code changes. This creates a reproducible record of the AI-assisted work that produced the PR.

2. **Run artifact** — include the `.ops/runs/<run_id>/last_run.json` file generated by Cursor during that session. Commit it to your feature branch at `.ops/runs/<run_id>/last_run.json` so reviewers can inspect the full audit trail.

3. **Tests** — any new or modified API route must be covered by tests. PRs that introduce untested routes will not be merged.
